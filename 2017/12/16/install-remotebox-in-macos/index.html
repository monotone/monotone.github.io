macos下安装remotebox

发表于 2017-12-16 | 分类于 技术 |

要在centos命令行系统中安装一个特定系统的虚拟机来使用，之前没通过命令行装过，又赶时间要用，所以就想抄捷径，弄个远程GUI管理工具来用。就搜索到了remotebox，这里记下来。

不过事实证明，走捷径不知道原理，出了大问题真的难以解决。

安装VirtualBox

参考官方文档：

https://www.virtualbox.org/wiki/Linux_Downloads

配置yum源：

vim /etc/yum.repos.d/virtualbox.repo

然后用yum直接安装即可：

yum install VirtualBox-5.2

安装扩展包

配置好VirtualBox

出现如上错误: yum install kernel-devel-uname -r，如果实际安装的版本比当前的内核版本更新，建议直接更新内核吧。更容易解决。

增加一个用户（不能用root，死活登录上的）,就算登录上了，创建虚拟机也会失败，因为默认虚拟磁盘是创建在用户主目录下的。

编辑配置文件：

确保日志文件存在并且属于对应的用户

到这里，就可以尝试去启动了

NOTE：

貌似root用户启动的vboxwebsrv，远程使用root连接不上，我是另外弄了一个属于root组的用户远程登录的，就成功了。

从官网网站下载对应VirtualBox版本的RemoteBox包

下载地址为：http://remotebox.knobgoblin.org.uk/downloads.cgi

安装RemoteBox

xquartz安装

从 http://xquartz.macosforge.org/landing 下载安装XQuartz

macports安装

从http://www.macports.org下载系统的macports版本，注意确保安装了xcode的命令行工具集。我这里是直接下载二进制安装包安装。

安装完成后，如果运行port命令提示找不到，那么确保$PATH有 /opt/local/bin:/opt/local/sbin: 即可。

然后执行sudo port install perl5 p5-gtk2 p5-soap-lite freerdp

修改remotebox启动脚本

进入remotebox文件夹，修改remotebox脚本，第一行#!/usr/bin/env perl为#!/opt/local/bin/perl5。然后直接执行这儿脚本，没有问题的话remotebox算是安装完了。

# macos # remotebox
